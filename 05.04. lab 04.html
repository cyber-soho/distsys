A.1
Process Snapshot with ps aux
▼

The ps command (historically: process status) reports a snapshot of running processes. The flags aux combine: a = all users, u = user-oriented format, x = include processes without a TTY (TeleTYpewriter).

# Step 1: Full process snapshot
ps aux

# Step 2: Sort by CPU consumption, top 20
ps aux --sort=-%cpu | head -20

# Step 3: Count processes by state (from STAT column)
ps aux | awk 'NR>1 {print $8}' | sort | uniq -c | sort -rn
ColumnMeaning
PIDProcess ID — unique identifier for this process
%CPUCPU usage percentage averaged over process lifetime
%MEMRSS (Resident Set Size) as % of total physical RAM
VSZVirtual Size in KB — total virtual address space claimed
RSSResident Set Size — physical RAM actually in use (KB)
STATProcess state (R/S/D/Z/T + modifier flags)
TTYControlling terminal; ? means none (daemon)
COMMANDThe executable and its arguments

A.2
Process Tree with pstree
▼

pstree reveals the parent–child hierarchy of processes — the family tree rooted at PID 1 (systemd). This is essential for understanding how the init system manages all services.

# Display full tree with PIDs (Process IDs) and user names
pstree -p -u

# Find the SSH (Secure Shell) daemon subtree
pstree -p -u | grep -A 5 -B 2 "sshd"

# Trace a specific process's full ancestor chain (replace 'nginx' as needed)
pstree -s -p $(pgrep nginx | head -1)

A.3
Deep Dive into the /proc Virtual Filesystem
▼

The /proc filesystem is not stored on disk. It is a virtual filesystem the Linux kernel exposes in real-time. Every numbered directory corresponds to a live process. This is how ps, top, and htop gather their data.

# Find the SSH daemon's PID (Process ID)
SSHD_PID=$(pgrep -x sshd | head -1)
echo "sshd PID is: $SSHD_PID"

# Explore its /proc directory
ls -la /proc/$SSHD_PID/

# Read the launch command (null-byte separated → replace with spaces)
cat /proc/$SSHD_PID/cmdline | tr '\0' ' ' && echo

# Read the process status (memory, threads, UID — User ID)
cat /proc/$SSHD_PID/status

# Count open FDs (File Descriptors)
ls /proc/$SSHD_PID/fd | wc -l

# Check cgroup (Control Group) membership — systemd's tracking
cat /proc/$SSHD_PID/cgroup

# System-wide information from /proc
cat /proc/uptime
cat /proc/meminfo | grep -E "MemTotal|MemAvailable|Cached"
cat /proc/mounts | column -t
/proc/$PID/ EntryContents
cmdlineThe exact command that launched this process (null-byte separated)
statusHuman-readable status: memory, threads, UIDs (User IDs)
fd/All open File Descriptors — files, sockets, pipes
mapsVirtual memory map — shows loaded shared libraries
cgroupControl Group membership (systemd tracking & resource limits)
net/Network state from this process's namespace
A.4
Sending Signals — IPC (Inter-Process Communication) for Process Control
▼
# List all available signals
kill -l

# Create a test process
sleep 600 &
SLEEP_PID=$!

# Send SIGTERM (signal 15) — graceful termination request
kill -SIGTERM $SLEEP_PID

# Verify it terminated
sleep 1 && ps -p $SLEEP_PID && echo "Still running" || echo "Process terminated"
SignalNumberBehaviourUse Case
SIGTERM15Catchable — asks process to exit gracefullyDefault for kill
SIGKILL9Cannot be caught or ignored — kernel forces terminationLast resort only
SIGHUP1Historically: terminal hangup; conventionally: reload confignginx -s reload
SIGINT2Keyboard interrupt (Ctrl+C)Interactive stop
SIGSTOP19Cannot be caught — pauses processDebugging
SIGCONT18Resumes a stopped processAfter SIGSTOP
Important: SIGKILL (kill -9)

SIGKILL bypasses the process entirely and cannot be caught, blocked, or ignored. It forces immediate kernel-level termination. Use it as a last resort — it does not allow cleanup, which can leave databases or log files in a corrupted state.

▼

[Unit]
Description=Human-readable description of what this unit does
Documentation=https://link-to-docs.example.com
After=network-online.target    # Start AFTER network has IP addresses
Wants=network-online.target    # Soft dependency — if network fails, still try
Requires=some.service          # Hard dependency — if this fails, we fail too

[Service]
Type=simple                    # ExecStart process IS the service
User=infrascale                # Never run as root
ExecStart=/usr/bin/myapp
Restart=on-failure             # Auto-restart on non-zero exit or signal
RestartSec=5s                  # Wait 5 seconds before restarting
StartLimitInterval=30s         # Burst window
StartLimitBurst=5              # Max restarts in the window

[Install]
WantedBy=multi-user.target     # Enable for "server ready" state
Type=Meaning
simpleExecStart process IS the service (default, most common)
forkingService forks; parent exits (traditional Unix daemons)
notifyService sends sd_notify() when truly ready
oneshotRuns once and exits; systemd waits for completion
idleLike simple, but waits until all active jobs complete
B.2
Write the InfraScale Health Service
▼
Golden Rule

Always test a service manually before writing a unit file. Never write a systemd unit for something you haven't verified works by hand.

# Step 1: Create directory structure
sudo mkdir -p /opt/infrascale/bin /opt/infrascale/logs

# Step 2: Write the health check script (Python — feel free to rewrite in Go, Rust, etc.)
sudo tee /opt/infrascale/bin/healthcheck.py << 'EOF'
#!/usr/bin/env python3
from http.server import HTTPServer, BaseHTTPRequestHandler
import json, socket, datetime, os

class HealthHandler(BaseHTTPRequestHandler):
    def do_GET(self):
        if self.path == '/health':
            payload = {
                "status": "ok",
                "hostname": socket.getfqdn(),
                "timestamp": datetime.datetime.utcnow().isoformat() + "Z",
                "service": "infrascale-health",
                "version": os.getenv("INFRASCALE_VERSION", "1.0.0")
            }
            body = json.dumps(payload, indent=2).encode()
            self.send_response(200)
            self.send_header("Content-Type", "application/json")
            self.send_header("Content-Length", str(len(body)))
            self.end_headers()
            self.wfile.write(body)
        else:
            self.send_error(404)
    def log_message(self, format, *args):
        print(f"[{datetime.datetime.utcnow().isoformat()}] {format % args}", flush=True)

if __name__ == "__main__":
    port = int(os.getenv("HEALTH_PORT", "8080"))
    server = HTTPServer(("0.0.0.0", port), HealthHandler)
    print(f"Starting on port {port}", flush=True)
    server.serve_forever()
EOF
sudo chmod +x /opt/infrascale/bin/healthcheck.py

# Step 3: Test manually FIRST
python3 /opt/infrascale/bin/healthcheck.py &
sleep 1 && curl -s http://localhost:8080/health | python3 -m json.tool
kill $!

# Step 4: Create dedicated system user (security best practice)
sudo useradd --system --no-create-home --shell /usr/sbin/nologin infrascale
sudo chown -R infrascale:infrascale /opt/infrascale
# Step 5: Write the unit file
sudo tee /etc/systemd/system/infrascale-health.service << 'EOF'
[Unit]
Description=InfraScale Health Check HTTP (Hypertext Transfer Protocol) Endpoint
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=infrascale
Group=infrascale
WorkingDirectory=/opt/infrascale
Environment="HEALTH_PORT=8080"
Environment="INFRASCALE_VERSION=1.0.0"
ExecStart=/usr/bin/python3 /opt/infrascale/bin/healthcheck.py
Restart=on-failure
RestartSec=5s
StartLimitInterval=30s
StartLimitBurst=5
PrivateTmp=yes
NoNewPrivileges=yes
ProtectSystem=strict
ReadWritePaths=/opt/infrascale/logs
SyslogIdentifier=infrascale-health

[Install]
WantedBy=multi-user.target
EOF

# Step 6: Enable and start
sudo systemctl daemon-reload
sudo systemctl enable --now infrascale-health.service
sudo systemctl status infrascale-health.service
curl -s http://localhost:8080/health | python3 -m json.tool

B.4
Restart Policies — Intentional Failure Testing
▼
# Experiment 1: Single crash → auto-restart
HEALTH_PID=$(systemctl show -p MainPID --value infrascale-health.service)
sudo kill -9 $HEALTH_PID
sleep 2
sudo systemctl status infrascale-health.service
# Expect: new PID, Active: active (running)

# Experiment 2: Burst crash → systemd gives up
for i in {1..6}; do
    PID=$(systemctl show -p MainPID --value infrascale-health.service)
    echo "Crash attempt $i — killing PID $PID"
    sudo kill -9 $PID 2>/dev/null
    sleep 4
done
# Expect: Active: failed — "Start request repeated too quickly"

# Recovery after burst failure
sudo systemctl reset-failed infrascale-health.service
sudo systemctl start infrascale-health.service

C.2
systemd Timers — The Modern Alternative
▼

cron vs. systemd timers

⏰ cron

  • Simple, universally available
  • Logging to syslog only — hard to filter
  • Missed runs are lost forever
  • No dependency awareness
  • No resource limits
  • Per-second scheduling impossible

⚡ systemd timer

  • Full journald (structured log) integration
  • Missed-run recovery (Persistent=true)
  • Dependency on other units
  • Full resource limits via service unit
  • Per-second scheduling (OnCalendar=*:*:0/30)
  • Boot-relative scheduling (OnBootSec=5min)
# The paired service unit (does the actual work)
sudo tee /etc/systemd/system/infrascale-cleanup.service << 'EOF'
[Unit]
Description=InfraScale Log Cleanup Job

[Service]
Type=oneshot
User=infrascale
ExecStart=/usr/bin/find /opt/infrascale/logs -name "*.log" -mtime +7 -delete
ExecStart=/usr/bin/journalctl --vacuum-time=30d
Nice=10
SyslogIdentifier=infrascale-cleanup
EOF

# The timer unit (controls when to run the service)
sudo tee /etc/systemd/system/infrascale-cleanup.timer << 'EOF'
[Unit]
Description=InfraScale Log Cleanup — Daily at 02:30

[Timer]
OnCalendar=*-*-* 02:30:00
Persistent=true
RandomizedDelaySec=60
Unit=infrascale-cleanup.service

[Install]
WantedBy=timers.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable --now infrascale-cleanup.timer
systemctl list-timers infrascale-cleanup.timer

Efficient Navigation — vim + fzf (Fuzzy Finder) + rg (ripgrep)
▼
🔬 Tool Freedom

This section uses vim + fzf + rg as one proven combination. You might prefer neovim + telescope.nvim, helix editor, or emacs with helm. Research, experiment, and advocate for what works best for your workflow. Even better: build your own!

# Install tools
sudo dnf install -y vim fzf ripgrep   # Fedora/RHEL/CentOS
sudo apt install -y vim fzf ripgrep   # Debian/Ubuntu

# fzf (Fuzzy Finder): interactive file selection → open in vim
vim $(ls /etc/systemd/system/*.service | fzf --preview 'cat {}')

# rg (ripgrep): fast recursive content search
rg "ExecStart" /etc/systemd/system/ -C 3       # 3 lines context
rg "Restart=" /etc/systemd/system/ -c          # count per file
rg -l "WantedBy=multi-user.target" /etc/systemd/system/  # filenames only

# POWER COMBO: rg + fzf + vim — interactive grep-to-editor jump
# Add this function to your ~/.bashrc
vimgrep() {
    local result
    result=$(rg --line-number --color=always "$@" /etc/systemd/system/ \
        | fzf --ansi \
              --delimiter=':' \
              --preview='bat --color=always --highlight-line {2} {1} 2>/dev/null || cat {1}' \
              --preview-window='right:60%')
    if [[ -n "$result" ]]; then
        local file line
        file=$(echo "$result" | cut -d: -f1)
        line=$(echo "$result" | cut -d: -f2)
        vim +"${line}" "$file"
    fi
}

source ~/.bashrc
vimgrep "ExecStart"
