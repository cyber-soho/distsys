Continuous Integration (CI)

The practice of merging, building, and testing code on every commit to a repository.

Continuous Delivery vs. Continuous Deployment

In Delivery, the validated build is ready for production but requires manual approval. In Deployment, it automatically reaches production.

Shift-left testing

The practice of moving testing, quality, and security checks earlier in the software development lifecycle.

Core idea of GitOps

The cluster pulls from Git. Git is the absolute source of truth, and the cluster continually reflects it.

The Four GitOps Principles

Declarative, Versioned & immutable, Pulled automatically, and Continuously reconciled.

Push vs. Pull deployment models

In Push (classic CI), the CI pipeline holds cluster credentials. In Pull (GitOps), the cluster controller initiates the fetch, keeping credentials out of CI.

Drift correction in GitOps

Automatic. The controller continuously reconciles the cluster state against the Git repository.

Argo CD vs Flux CD origin

Argo CD is Intuit-born and dashboard-rich; Flux CD is Weaveworks-born and GitOps-native.

Fundamental architecture of Tekton

The pipeline engine is Kubernetes itself. Each pipeline step runs as a Pod, and definitions are standard Kubernetes CRDs.

Tekton Primitive: Task

One unit of work, executed as a Pod with one or more sequential steps.

Tekton Primitive: PipelineRun

An active execution of a Pipeline; it is the object that actually creates the Pods to do the work.

Tekton Primitive: Workspace

Shared storage between Tasks, usually backed by Persistent Volume Claims (PVCs).

Tekton Primitive: Trigger

Listens for external events (like webhooks) and automatically starts PipelineRuns.

Rolling Deployment

Replaces Pods one (or N) at a time, resulting in mixed versions running simultaneously during the rollout.

Blue-Green Deployment

Maintains two complete environments. Cutover happens instantly by flipping the load balancer, allowing for instant rollbacks.

Major downside of Blue-Green deployments

It costs roughly double during the cutover window because two full environments must run simultaneously.

Canary Deployment

A gradual percentage-based rollout of traffic to a new version (e.g., 1%, then 10%, then 100%).

Recreate Deployment

Kills all old pods before starting the new ones. Best for version incompatibilities, but guarantees downtime.

Primary focus of Terraform

Infrastructure provisioning (creating VMs, VPCs, Firewalls) using declarative configuration.

Primary focus of Ansible

Configuration management (installing software and configuring the OS inside the provisioned machines).

Ansible execution method

Agentless execution, running commands remotely over SSH or WinRM.

Terraform State File

A file that tracks the current state of managed resources, allowing Terraform to know what needs to be created, updated, or deleted.

OpenTofu

An open-source fork of Terraform created by the Linux Foundation after HashiCorp moved to a Business Source License.

Idempotent Tasks (Ansible)

Tasks that are safe to run repeatedly. Running them multiple times produces the same result as running them once.

The Unified Principle of the Module

Every change is a pull request; every truth lives in Git; every system reconciles itself toward that truth.

1. A development team has automated their pipeline to the point where every validated build sits in a staging environment awaiting a manual approval click from a release manager before it goes live. Which practice does this workflow represent?

  • Continuous Deployment
  • Continuous Integration
  • Shift-left testing
  • Continuous Delivery
✅ Show Answer & Rationale

Correct Answer: Continuous Delivery.
Requiring a manual approval step before a validated build goes to production is the defining characteristic that separates this practice from fully automated deployments (Continuous Deployment).

2. A security team wants to catch code vulnerabilities as soon as developers commit their code, rather than waiting for a dedicated security scan in the staging environment days later. Which concept are they implementing?

  • Shift-left testing
  • Smoke testing
  • Continuous Deployment
  • Infrastructure as Code
✅ Show Answer & Rationale

Correct Answer: Shift-left testing.
Moving security and quality checks to the earliest possible stages of development prevents vulnerabilities from progressing further down the pipeline.

3. A junior administrator uses the command line to manually increase the number of replicas for a production deployment. Shortly after, the replicas automatically revert back to their original count. Which GitOps principle caused this behavior?

  • Declarative configuration
  • Versioned and immutable
  • Push deployment
  • Continuous reconciliation
✅ Show Answer & Rationale

Correct Answer: Continuous reconciliation.
GitOps controllers constantly monitor the cluster state and automatically overwrite any manual changes that do not match the declared state in the repository.

4. A security auditor is reviewing your deployment architecture and insists that the continuous integration server must not store any credentials that allow direct modification of the production cluster. Which deployment model satisfies this requirement?

  • A classic Push-based CI pipeline
  • A Blue-Green deployment strategy
  • A Pull-based GitOps model
  • A procedural configuration management model
✅ Show Answer & Rationale

Correct Answer: A Pull-based GitOps model.
The cluster controller reaches out to fetch configurations securely from within the cluster, eliminating the need to expose production credentials to an external build server.

5. You have defined a complex workflow for building and packaging an application in Tekton, but no pods are spinning up to actually perform the work. Which primitive must you create to trigger the execution?

  • Workspace
  • Trigger
  • PipelineRun
  • Task
✅ Show Answer & Rationale

Correct Answer: PipelineRun.
This object instructs Kubernetes to actually instantiate the required Pods and execute the defined workflow using specific inputs.

6. An engineering director wants to reduce infrastructure costs by eliminating the need for a dedicated, always-on continuous integration server, preferring a solution that scales dynamically using their existing Kubernetes compute capacity. Which tool is best suited for this?

  • Jenkins
  • Tekton
  • GitLab CI/CD
  • CircleCI
✅ Show Answer & Rationale

Correct Answer: Tekton.
Operating natively as a set of Kubernetes Custom Resource Definitions allows this engine to utilize the cluster's existing compute nodes and scale automatically.

7. An e-commerce site is launching a major overhaul of its payment gateway on Black Friday. They require zero downtime during the switch and the ability to instantly revert to the old version if errors occur, regardless of infrastructure costs. Which deployment strategy should they use?

  • Rolling deployment
  • Canary deployment
  • Blue-Green deployment
  • Recreate deployment
✅ Show Answer & Rationale

Correct Answer: Blue-Green deployment.
Maintaining a completely separate, pre-warmed environment allows traffic to be flipped instantly, ensuring zero downtime and immediate reversion capabilities despite doubled infrastructure costs.

8. A legacy application is being updated, and the new version cannot run concurrently with the old version due to strict database schema locks. Which strategy must the team utilize, even though it will cause a brief outage?

  • Rolling deployment
  • Recreate deployment
  • Blue-Green deployment
  • Shadow deployment
✅ Show Answer & Rationale

Correct Answer: Recreate deployment.
Completely terminating the old application instances before starting the new ones guarantees that no two incompatible versions are attempting to access locked resources simultaneously.

9. A cloud engineering team needs to provision 50 new virtual machines across three cloud regions, and then install and configure a proprietary web server application inside each of those operating systems. Which combination of tools fits this workflow best?

  • Ansible for the virtual machines, Terraform for the web server
  • OpenTofu for both tasks
  • Terraform for the virtual machines, Ansible for the web server
  • Argo CD for the virtual machines, Tekton for the web server
✅ Show Answer & Rationale

Correct Answer: Terraform for the virtual machines, Ansible for the web server.
HCL is highly optimized for declarative cloud resource provisioning, while agentless YAML playbooks excel at configuring the internal state of those operating systems.

10. A systems administrator accidentally runs an Ansible playbook three times in a row against a fleet of servers. The servers remain in the exact desired state without any duplicate configurations or errors. Which property of Ansible tasks makes this possible?

  • Declarative syntax
  • Drift detection
  • Continuous reconciliation
  • Idempotency
✅ Show Answer & Rationale

Correct Answer: Idempotency.
Designing tasks so they only apply changes if the current state differs from the desired state ensures that repeated executions remain perfectly safe and predictable.
