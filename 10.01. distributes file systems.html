Module 3: Protocols in Practice

We move from theory to practical implementations. We examine NFS, the long-standing standard for Unix file sharing, and HDFS, designed for petabyte-scale big data analytics.

Network File System (NFS)

Developed in 1984, NFS allows clients to mount remote directories as if they were local. Current standard is NFSv4 (Stateful, TCP-only, Kerberos security).

Server: Exporting (/etc/exports)

/data/shared 192.168.1.0/24(rw,sync)
/data/readonly *(ro,root_squash)

rw=read/write, ro=read-only, sync=safe writes, root_squash=security mapping.

Client: Mounting

mount -t nfs -o rw,hard,intr server:/share /mnt/nfs

hard=retry indefinitely (safe), intr=allow cancellation, timeo=timeout.

Hadoop Distributed File System (HDFS)

Optimized for "write once, read many times" at petabyte scale across commodity hardware. Utilizes a strict Master-Slave architecture.

🧠 NameNode (Master)

  • Manages namespace in RAM
  • FsImage (snapshot) + EditLog (changes)
  • NEVER touches actual data

🗄️ DataNodes (Slaves) & Blocks

Files are split into 128 MB blocks. Default Replication Factor (RF) = 3.

Module 4: Resilience & Management

Hardware fails constantly. DFS handles this via replication and checksums. Finally, we look at admin tools like autofs for managing massive distributed environments efficiently.

⚠️ Fault Tolerance Mechanisms

  • 💓
    Heartbeats (Detection) DataNodes ping NameNode every 3s. No ping for 10m = Dead Node → Trigger re-replication.
  • ✓
    Checksums (Integrity) Computed on write, verified on read. Mismatch = Corrupt → Fetch from healthy replica.
  • 🔄
    Replication (Recovery) RF=3 means 200% storage overhead, but guarantees survival against rack failures. (Erasure coding in Hadoop 3.x reduces this to ~50%).

autofs: Scale Management

Managing hundreds of static NFS mounts in /etc/fstab is unscalable. autofs solves this.

  • On-demand: Mounts only when accessed.
  • Auto-unmount: Disconnects idle shares (5 min default).
  • Graceful: Workstations boot even if NFS server is down.
# Master Map (/etc/auto.master)
/home    /etc/auto.home   --timeout=300
# Map File (/etc/auto.home) using wildcard
*        -rw,hard,intr  server:/exports/home/&
