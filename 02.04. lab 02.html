Exercise 1 & 2: DNS Infrastructure

In this section, you will explore DNS query tools and then implement a fully functional BIND9 server. Use the interactive simulators below to understand how the configuration files map to your network design.

Authoritative Server

The server holding the original zone data. The source of truth for your domain.

Forward Zone

Maps Hostnames → IP Addresses (A Records). "Where is mail.google.com?"

Reverse Zone

Maps IP Addresses → Hostnames (PTR Records). "Who is 192.168.1.1?"

Recursive Resolver

Queries other servers on behalf of the client to find the answer.

Troubleshooting & Automation

                    
Task 4 requires automating tasks and validating the setup. Use this guide to resolve common errors.


                    
                    

                        

                            
                                DNS: NXDOMAIN / Dig Timeouts
                                
                                    
                                
                            
                            

                                
    
                                    
  • Syntax Errors: Run named-checkconf and named-checkzone. A missing semicolon is fatal.
    • 
                                    
  • Firewall: Ensure port 53 (TCP/UDP) is open. sudo ufw allow 53.
    • 
                                    
  • Serial Number: Did you increment the serial in the zone file after editing? Restart BIND after.
    • 
                                

                            

                        


                        

                            
                                LDAP: Invalid Credentials / Login Failures
                                
                                    
                                
                            
                            

                                
    
                                    
  • Bind DN: Verify you are using the correct admin DN (e.g., cn=admin,dc=infrascale,dc=local).
    • 
                                    
  • PAM Config: On the client, run sudo pam-auth-update and ensure LDAP is selected.
    • 
                                    
  • Home Directory: If login works but closes immediately, enable pam_mkhomedir to create user home folders automatically.
    •
