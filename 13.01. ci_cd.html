Explore the automated chain that turns a commit into a deployment. Without it, you have a release ritual; with it, you have an industrial process.

Merge, build, and test on every commit to the repository.

Lesson 2: GitOps Workflow

The paradigm shift: The cluster pulls from Git. Git is the source of truth. Reality follows Git, not the other way around.

The Four Principles

1. Declarative

Desired state described entirely in code.

2. Versioned & Immutable

Every change is a tracked Git commit.

3. Pulled Automatically

Controllers fetch and apply configurations.

4. Continuously Reconciled

Configuration drift is detected and corrected.

Lesson 4: Deployment Strategies

Match the strategy to the application's tolerance for mixed-version operation and the cost of an outage. There is no universal best answer.

Strategy Trade-offs Analysis

This chart visualizes the matrix of deployment strategies based on their inherent costs, downtime risk, and how fast you can rollback if things go wrong. Lower scores are better for Downtime/Cost, higher scores are better for Rollback speed.

🔄 Rolling Deployment (The Default)

  • • Replaces Pods one (or N) at a time.
  • • Mixed versions exist during rollout.
  • • Controlled by maxSurge and maxUnavailable.
  • • Best for: Stateless services, low-risk changes.

🔵🟢 Blue-Green Deployment

  • • Two complete environments (Blue=current, Green=new).
  • • Cutover happens instantly by flipping the load balancer/Ingress.
  • • Instant rollback capability.
  • • Downside: Costs ~2x during the cutover window.
  • • Best for: Critical correctness, big major releases.

Other Noted Strategies

Canary: Gradual % rollout (1% → 10% → 100%).
A/B Testing: Variant routing for product experiments.
Shadow: Copy of traffic, discarded responses.
Recreate: Kill all, start all (causes downtime).

Lesson 5: Infrastructure as Code (IaC)

Cloud accounts and OS configurations are code. They go in Git. They get reviewed. Snowflake servers are a 2010s problem.

Terraform / OpenTofu

Provisioning
  • Type: Declarative (HCL)
  • Mechanism: State file tracks managed resources.
  • Safety: plan previews changes before applying.
  • Reusability: Uses Modules for building blocks.
  • Scope: Multi-cloud (~3,000 providers).
*2026 Footnote: OpenTofu is the open-source fork (Linux Foundation) drop-in replacement after HashiCorp's BSL licensing change.

Ansible

Configuration
  • Type: Procedural / Hybrid (YAML playbooks)
  • Mechanism: Agentless (runs over SSH or WinRM).
  • Safety: Idempotent tasks (safe to run repeatedly without breaking things).
  • Targeting: Inventory-driven.
  • Scope: Best for what runs inside the provisioned machines.
